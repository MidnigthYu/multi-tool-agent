# Release 文案模板

从 v0.7.0 起，所有 GitHub Release 统一使用以下规范，确保版本文案专业一致。

---

## 一、标题格式

```
vX.Y.Z-模块标识 | 核心能力一句话定位
```

- **模块标识**：英文简写，与 git tag 完全对应（如 `weekly`、`memory`、`rag`）
- **后半句**：陈述句，清晰说明本次版本的核心交付

---

## 二、正文固定三段式结构

| 板块 | 用途 | 列表格式 | 建议条数 |
|------|------|----------|----------|
| **核心亮点** | 高度提炼版本价值，给读者快速建立认知 | 无序列表（`-`） | 4 条以内 |
| **功能与优化** | 逐条展开具体功能、修复、优化项 | 有序列表（`1.`） | 6–8 条 |
| **工程质量** | 统一展示测试、类型、规范、覆盖率门禁数据 | 无序列表（`-`） | 3–4 条 |

### 板块说明

**核心亮点** — 回答「读者最需要知道什么」：
- 本版本解决了什么问题
- 最核心的能力是什么
- 架构上有什么亮点
- 异常/边界场景如何兜底

**功能与优化** — 逐条展开，顺序按重要性排列：
- 新增功能在前，修复/优化在后
- 每条首句点明模块或改动对象
- 使用「新增」「修复」「统一」「优化」等动词开头

**工程质量** — 数据口径统一，不混用不同表述：
- 测试用例数量及通过状态
- 类型校验与代码规范状态
- 覆盖率数值及门禁标准

---

## 三、数据口径规范

以下表述为固定句式，全版本统一，不随意替换措辞：

| 维度 | 统一表述 |
|------|----------|
| 测试用例 | 全项目合计 XX 条单元测试，零失败 |
| 代码覆盖率 | 整体代码覆盖率 XX%，持续满足 ≥90% 归档门禁标准 |
| 静态类型 | mypy --strict 核心业务目录零类型错误 |
| 代码规范 | ruff 全量规范校验零告警 |

---

## 四、完整示例（v0.7.0-weekly）

### 标题

`v0.7.0-weekly | 结构化周报生成能力正式落地`

### 正文

```markdown
## 核心亮点
- 基于三级会话记忆的结构化周报生成能力，支持 Markdown / JSON 双格式输出
- 纯代码硬统计 + LLM 语义总结分层实现，数据准确可追溯，语义总结自然完整
- 最小侵入式接入主链路，仅新增工具分支，不修改核心调度逻辑与历史功能
- 空会话、超长会话、LLM 故障、参数异常四类场景全兜底，异常内部闭环不中断对话

## 功能与优化
1. 新增 `generate_weekly_report` 函数式工具，遵循 ToolRegistry 注册规范，`get_agent` 初始化自动幂等注册
2. 调度节点从 AgentState 实时注入会话数据：消息总数、工具调用记录、会话时长、中期记忆摘要，工具层保持无状态
3. 工具调用统计、会话时长由代码硬计算，LLM 仅负责语义总结，避免数值偏差
4. 内置 3000 字符超长内容自动截断机制，保留核心摘要与关键工具记录，避免上下文溢出
5. 四层降级兜底：空会话友好提示、参数非法自动回退、LLM 故障返回纯统计表格、超长内容智能截断
6. 修复调度节点统计数据空注入问题，工具使用统计与会话时长均从运行状态实时读取
7. 统一 config 目录子模块导出风格，代码规范与全项目对齐

## 工程质量
- 新增 5 条周报工具专项单元测试，覆盖正常生成、空会话、非法格式、LLM 故障、超长内容全场景
- 全项目合计 312 条单元测试，零失败
- mypy --strict 核心业务目录零类型错误，ruff 全量规范校验零告警
- 整体代码覆盖率 91.38%，持续满足 ≥90% 归档门禁标准
```
